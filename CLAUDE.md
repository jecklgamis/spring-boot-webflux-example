## Build and Run

```bash
# Build the JAR (also generates keystore if needed)
./mvnw clean package

# Run locally
java -jar target/spring-boot-webflux-example.jar

# Build and run via Docker
make dist image run

# Generate TLS keystore (required before first Docker build)
./generate-keystore.sh
```

## Testing

```bash
# Run all tests
./mvnw test

# Run a single test class
./mvnw test -Dtest=KeyValueStoreHandlerTest

# Run a single test method
./mvnw test -Dtest=KeyValueStoreHandlerTest#testStoreAndRetrieve
```

Tests use `@SpringBootTest(webEnvironment = RANDOM_PORT)` with `WebTestClient` for integration testing against a live server context.

## Architecture

This is a Spring Boot WebFlux app using **functional endpoints** (not annotation-based controllers). The key design pattern is:

- **`RouterConfig`** — defines all routes using `RouterFunctions.route()`, mapping HTTP verbs/paths to handler methods
- **Handlers** (`handler/`) — `@Component` classes with methods of signature `Mono<ServerResponse> method(ServerRequest)`. Four handlers exist: `RootHandler`, `FileUploadHandler`, `KeyValueStoreHandler`, `ProbeHandler`
- **Services** (`service/`) — `KeyValueStoreService` interface with `InMemoryKeyValueStoreServiceImpl` as the only implementation (in-memory, non-persistent)
- **`ExampleWebClient`** — demonstrates WebClient usage patterns

All handler and service methods return reactive types (`Mono`/`Flux`) — avoid blocking calls.

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/` | Root |
| POST | `/upload` | Multipart file upload (saves to CWD with UUID prefix) |
| POST | `/kv/{key}` | Store key-value (body is plain text value) |
| GET | `/kv/{key}` | Retrieve value (404 if not found) |
| DELETE | `/kv/{key}` | Delete key |
| GET | `/probe/live` | Liveness probe |
| GET | `/probe/ready` | Readiness probe |
| GET | `/actuator/*` | Spring Boot Actuator (health, metrics, etc.) |

Server runs on port 8080 (HTTP) and 8443 (HTTPS). TLS uses `src/main/resources/keystore.pfx` (generated by `generate-keystore.sh`).
